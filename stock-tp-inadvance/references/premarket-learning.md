# 早盘模型的学习与进化（premarket-learning）

模块⑤：让 skill"越用越懂你、越用越准"。核心是**每天记录预测 → 收盘后回填实际 →
统计命中率 → 有纪律地微调权重与口径**。第一版为轻量版（人工触发复盘），后续可挂 /loop 自动化。

---

## 一、预测日志（自动）

每次跑 `premarket.py` 都会向 `reports/premarket_journal.jsonl` 追加一条：
```
{date, code, premarket_score, bias, pred_open, pred_high, pred_low, trend_label, weights,
 actual_open, actual_high, actual_low, actual_close, hit_high_err_pct, hit_low_err_pct, reviewed}
```
`actual_*` 与误差字段初始为 null，**收盘后回填**。

## 二、收盘复盘（每天，5 分钟）

1. 取当日实际 开/高/低/收（`python fetch_market.py <代码>` 的最新一根，或手填）。
2. 回填到日志对应记录，并算：
   - `hit_high_err_pct = (pred_high − actual_high)/actual_high ×100`
   - `hit_low_err_pct  = (pred_low  − actual_low )/actual_low  ×100`
   - 方向是否对：实际收盘相对昨收方向，与 bias 符号是否一致。
3. 记一句**定性归因**：今天哪个因子帮了/坑了？（如"隔夜半导体大涨但个股高开低走，隔夜因子高估"）

> Claude 可代做：读 `--review` 输出 → 调 `fetch_market` 取实际 → 计算误差 → 写回日志 → 给出归因。

## 三、统计与调参纪律（每周或满 10 条样本）

统计最近 N 条：
- **高/低点平均绝对误差**（越小越好）；
- **方向命中率**（bias 符号 vs 实际涨跌）；
- **分因子贡献复盘**：把每天"哪个因子方向对"累计，估算各因子的"胜率/IC"。

调参纪律（**防过拟合，照做**）：
1. 一次只调一个口径或一个权重；
2. 单次权重调整幅度 ≤ 0.03；
3. 永远保留"默认权重"一组做对照（A/B）；
4. 样本 < 10 不调参；连续 2 次复盘指向同一问题才动手；
5. 调整记录写进日志或单独 `reports/tuning-log.md`，可回滚。

**典型修正**：
| 复盘现象 | 调整方向 |
|---------|---------|
| 隔夜强但个股不跟，隔夜屡屡高估 | 调低 `W_PM_OVERNIGHT`，或在 linkage.json 降低该股 overnight_proxies 权重 |
| 同业齐涨该股必跟，联动很准 | 调高 `W_PM_LINKAGE` |
| 主力净流入次日多兑现 | 调高 `W_PM_CAPITAL` |
| 预测振幅普遍偏大/偏小 | 调整 framework 里 ATR 波动系数(0.9+0.4×|bias|) |

## 四、了解你的风险偏好（账户层）

结合持仓输入（买入价/股数）长期积累，Claude 应逐步总结并在 `reports/` 留档：
- 你倾向追高还是低吸？历史套牢多发生在什么情形（如"高位听消息买入")？
- 你能接受的单日回撤 / 止损纪律是否执行到位？
- 据此**个性化建议口吻**：对你这种"怕高位被套"的偏好，默认更强调"不追高、以预测低点低吸、套牢反弹减仓"。

## 五、了解中国股市的方向（中观）

定期（可借 `stock-tp` 的新闻/政策检索）更新对当前市场阶段的判断（牛/熊/震荡、主线板块轮动），
写入 `reports/market-regime.md`，作为早盘 bias 的背景修正：
- 上升浪：放大偏多信号、及时跟进强势股；
- 波浪底部/下行浪：收紧仓位、只做超跌反弹、严格止损，避开"接飞刀"。

> 进化 = 数据(日志) + 纪律(小步调参) + 归因(定性) + 你的偏好画像，四者叠加。不要靠单日结果大改权重。
