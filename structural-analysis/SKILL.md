# 既有建筑加固方案结构合理性分析 Skill

## 适用场景

对既有混凝土框架建筑的**加固 / 改造方案**做**定量合理性论证**，尤其是荷载大幅增加（如普通办公改为**金库 / 档案库 / 重型库房**，活载从 4 kN/m² 跃升到 12~40 kN/m²）的场景。核心问题往往是：**"加固"是否还成立，还是应当"拆除重建"**。最终产出一份结构严谨、图文并茂的**中文 PDF 研究报告**。

典型触发：
- "活载从 4 加到 40，原柱够不够？要不要拆了重建？"
- "帮我论证这个加固方案合不合理 / 出一份结构专篇分析报告"
- "消能减震能不能解决这个承载力问题？"
- 上传 `xxx方案数据.docx` + 既有图纸 PDF，要求复核并出报告

## 依据规范

| 规范 | 用途 |
|------|------|
| **GB 50010-2010** | 混凝土结构设计标准（承载力、配筋） |
| **GB 50011-2010** | 建筑抗震设计规范（轴压比限值、地震作用） |
| **GB 55008-2021** | 混凝土结构通用规范（强制性工程规范） |
| **GB 50367** | 混凝土结构加固设计规范（二次受力、加固材料利用率） |

> 抗震烈度变化（如 7 度→8 度）、抗震等级变化（三级→二级）会显著放大地震作用与轴压比要求，必须在分析中显式处理。

## 分析方法论（九步）

按 `scripts/analysis.py` 的结构逐项定量计算，**每一项都给出"原设计 vs 改造后"的对比和倍数**：

1. **原结构基本参数** — 柱网、层数层高、材料(C40/HRB400)、典型柱/梁截面、负荷面积。
2. **荷载对比** — 恒载+0.5活载 = 重力荷载代表值；算出改造后相对原设计的**荷载倍数**。
3. **柱轴压比验算**（GB50011 表6.3.6）— `N/(fc·A)` 对比限值（二级0.75 / 三级0.85）；超限时反算**需要的截面**与倍数，并验证常见放大截面（如 1200×1200）是否仍勉强。
4. **地震力对比** — 底部剪力 `V = αmax·G`（7度0.08 / 8度0.16）；给出地震力**放大倍数**。
5. **基础/桩基承载力** — 标准组合轴力 / 桩群承载力 = 利用率；超限时算**需补桩数**（含在既有地下室补桩的施工难度提示）。
6. **梁承载力与净高** — 弯矩 `M = q·L²/8` 对比估算抗弯承载力 `Mu`；反算所需梁高，校核**梁下净高**是否满足使用要求（如金库 ≥3000mm）。
7. **二次受力 / 应力滞后** — 加固材料只能分担"加固后新增荷载"，原构件已"带伤工作"；量化恒载应力占比与加固材料有效利用率(60~80%)。
8. **消能减震局限性** — 阻尼器只降水平地震响应、不提竖向承载力；说明"构造措施降一度"≠ 承载力解决。
9. **综合结论与建议** — 汇总表 + 是否应转向"拆除重建"的判断。

### 关键计算（摘自 analysis.py）

```python
# 重力荷载代表值
gravity_repr = dead_load + 0.5 * live_load

# 柱轴压比（设计组合）
N = (1.2*dead + 1.4*live) * trib_area * n_floors        # kN
axial_ratio = N*1000 / (fc * col_area)                  # 对比限值 0.75/0.85

# 满足轴压比所需柱截面
required_area = N*1000 / (fc * axial_ratio_limit)

# 底部地震剪力
V = alpha_max * (gravity_repr * building_area * n_floors)

# 梁弯矩与所需梁高
M = (1.2*dead + 1.4*live) * span_y * span_x**2 / 8
h0_needed = sqrt(M*1e6 / (fy * rho * beam_b * 0.9))
```

## PDF 报告生成

报告用 **ReportLab + matplotlib** 生成（见 `scripts/generate_integrated_report.py`），要点：

- **中文字体**：matplotlib 与 ReportLab 都必须注册 Windows 中文字体（msyh.ttc / simhei.ttf / simsun.ttc）；ReportLab 用 `pdfmetrics.registerFont(TTFont(...))`，matplotlib 用 `FontProperties(fname=...)`，并设 `axes.unicode_minus=False`。
- **配色与样式**：深蓝主色 `#1a365d`、强调红 `#e53e3e`、成功绿 `#2f855a`；标题/正文/表格统一 `ParagraphStyle`；表格交替行底色。
- **示意图**（matplotlib 生成，存入 `report_images/`）：
  - `draw_plan_view_all_schemes` 各方案平面对比
  - `draw_section_comparison` / `draw_scheme3_section_detail` 剖面与加固详图
  - `draw_shear_force_comparison` 地震剪力对比柱状图
  - `draw_net_height_comparison` 梁下净高对比
  - `draw_comprehensive_radar` 多方案综合雷达图
- **结构**：封面 → 各章节(对应九步) → 对比表 → 综合结论。用 `SimpleDocTemplate(A4)` + `PageBreak` 组织。

## 脚本说明

| 脚本 | 作用 |
|------|------|
| `scripts/analysis.py` | 核心定量分析引擎（九步验算，控制台输出，作为方法论范本） |
| `scripts/generate_report.py` | 单方案结构分析 PDF 报告生成器 |
| `scripts/generate_report_bc50.py` | BC 楼半金库半办公方案专用报告 |
| `scripts/generate_integrated_report.py` | 整合多方案、含全部示意图的**综合研究报告**（最完整范本） |

> 这些脚本以"某银行金库改造"项目为**演示样例**，其中的柱网、荷载、截面等数值是该工程的具体取值。复用时按新项目参数替换顶部的常量即可。

## 使用流程

1. 从用户的方案数据（docx/PDF/口述）提取：柱网、层数层高、材料、截面、原/新恒活载、抗震烈度与等级、基础形式。
2. 以 `analysis.py` 为模板替换参数，跑出九步逐项验算与倍数对比。
3. 识别"致命项"（轴压比超限、净高不足、补桩量、二次受力）。
4. 用 `generate_integrated_report.py` 为模板生成中文 PDF（先画 matplotlib 示意图，再用 ReportLab 排版）。
5. 给出明确结论：加固可行性 vs 拆除重建建议，并量化支撑。

## 注意事项

- **务必显式核对抗震烈度/等级是否随改造提高**——这是地震力与轴压比成倍放大的根源。
- 加固分析必须包含**二次受力**判断，否则会高估加固后的安全裕度。
- 报告数值要可追溯：每个结论都附计算式与对比倍数，避免"定性空话"。
- 中文 PDF 在非 Windows 环境需改用对应中文字体路径（如 Linux 的文泉驿/Noto CJK）。
