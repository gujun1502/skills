# 招投标复盘与陷阱审计 Tender Bid Review

把一组招投标材料（公示→招标文件→答疑→投标响应→开标→评标→定标→中标公示）归纳成**一份决策级复盘报告**，核心交付四件事：

1. **全生命周期时间轴复盘**——从公示到最终结果，每个节点发生了什么、关键参数如何变化。
2. **投标中存在的问题**——文件矛盾、响应不当、被低估的工作量、踩雷点。
3. **我方资质与能力边界**——硬门槛是否过、软实力在哪个分项塌陷、洼地有多深。
4. **陷阱挖掘**——重点是"设在投标界限之外"的陷阱（评奖角度、业绩角度、利益交换角度），以及合理/不合理投标响应的判定。

> 核心心法：**真正决定胜负的陷阱，往往不在"能不能投"的硬门槛上，而在评分规则、奖项口径、业绩口径、定标机制里——它们合法、隐蔽，且是为某一类对手"量身定制"的规则性倾斜。** 你的任务是把这些"合法的偏向"挖出来，并诚实地告诉用户：哪些是规则使然（无法翻盘）、哪些是我方可补救的洼地。

## 工作流程

### 第 0 步（首要）：加载本单位画像 company-profile/

复盘/决策前**先读 `company-profile/` 文件夹**——这是用户预先填好的本单位资质、业绩、人员、奖项、关系、财务资料。一切"对照式"结论（能否过硬卡口、哪个分项是洼地、若参标排第几）都建立在它之上。

- 读 `company-profile/company-profile-template.md`（用户已填写的版本）及文件夹内任何 `业绩库.csv / 奖项库.csv / *.xlsx`。
- **若文件夹为空或关键项写"待补"**：先明确提示用户"要给出对照式结论，请先补全 company-profile/（尤其业绩库与奖项库口径）"，再决定是先给"通用版"复盘还是等用户补料。不要在缺画像时假装精确测算排位。
- 信息更新只改这里，下次复盘自动用最新画像。

### 第 0.5 步：清点材料、识别项目与阶段

扫描目标文件夹（默认当前目录），按项目和阶段给文件归类。一个文件夹常含**多个项目**、**多个版本**（流标版/重新招标版）、**多种角色文档**：

| 文档角色 | 典型文件名特征 | 用途 |
|---|---|---|
| 招标公告/公示 | 「…公示」「发布--…」「…公告」 | 项目参数、门槛、时间窗口 |
| 招标文件正文 | 「…招标文件」「正文」 | 评分办法、合同条款、废标条款（陷阱主矿区）|
| 答疑/澄清 | 「…答疑」「澄清」 | 业主对矛盾条款的官方口径，常修订陷阱 |
| 投标决策/风险报告 | 「…投标风险研究」「…决策报告」「…评判报告」 | 我方此前判断，复盘时校验其准确度 |
| 对手画像 | 「…对手画像」「竞争」「制胜策略」 | 利益交换/关系格局分析 |
| 中标候选人公示 | 「…中标候选人公示」「评标结果」 | 最终结果，逐项打分（评分反算的原料）|

先用 Read/Grep 把 `.md` 决策报告读完（信息密度最高），PDF 用 Read 的 `pages` 抽读关键章节（评分办法、废标条款、合同条款、公示打分表）。**不要假设只有一个项目**——逐文件确认归属。

### 第 1 步：还原全生命周期时间轴

按时间排出节点链，标注每个节点的关键参数变化（尤其**版本间 diff**——流标→重新招标常暴露业主的真实意图）：

```
公示发布 → 招标文件下载窗口 → 答疑/澄清截止 → 投标截止 → 开标(远程/不见面) → 评标(暗标/明标) → 定标 → 中标候选人公示 → 结果
```

对每个项目产出一张「关键参数表」：项目名、编号、业主、代理、规模、限价/固定价、保证金、资质门槛、业绩门槛、评标办法、开工节奏、各时间节点。**流标重招的，必须做新旧版逐条 diff**（见 references/case-library.md 兴业案例）。

### 第 2 步：评分结构还原与"评分反算取证"

如果有中标公示的逐项打分，**务必做评分反算**——这是判断"有没有猫腻"最硬的工具。方法见 `references/scoring-forensics.md`：

1. 拆出满分构成（商务 + 技术/暗标 + 报价）。
2. 用公示的逐项分**反算总分**，看是否分毫吻合（验证数据自洽）。
3. **报价分**：套评标基准价公式逐家代入，100% 应可复现——这块几乎不可能做手脚。
4. **技术暗标分**：看专家间是否"互相打架"。分歧大 = 健康盲评；高度雷同、整齐压制 = 可疑。
5. **商务分**：5 位专家给出**完全一致**的分，恰恰是"规则化客观分"的证据（数业绩、点奖项、算职称），不是舞弊。
6. **定位胜负手**：算出冠军对亚军的总分差，看这个差几乎全部来自哪一个分项——通常落在**商务标的奖项/业绩规则**上，而非暗标方案上。

结论分两层表述：**评标过程层面**（专家自由裁量）有没有舞弊；**招标规则层面**有没有"规则性倾斜"。后者才是常态陷阱。

### 第 3 步：我方资质与能力边界测绘

产出「能力覆盖度矩阵」，对每个分项判定：**硬门槛过不过 / 软实力满分还是洼地 / 洼地有多深 / 可否补救**。模板见 `references/capability-boundary.md`。

- **硬卡口（GO/NO-GO）**：资质、业绩门槛、设计负责人资格、信誉/失信、利害关系回避——缺一即废，逐条打勾。
- **软实力洼地**：奖项分、专项业绩分、答辩/镜头表现、关系基础——这些决定排位。
- **若我方参标排第几**：用第 2 步的尺子，做保守/中性/乐观三情景测算，诚实给出"大概率第几、要翻到第一需要同时发生哪几件低概率的事"。

### 第 4 步：陷阱挖掘（核心）

按 `references/trap-taxonomy.md` 的九类陷阱谱逐类排查。重点是用户强调的"**设在投标界限之外**"的三个角度：

- **评奖角度**：加分奖项是否锁死在某协会（如中国建筑装饰协会）？这等于给装饰名门的规则性倾斜，建筑学会/行业奖出身的院进不去。
- **业绩角度**：是否存在"双口径"（资格审查口径松、定标评分口径严）？专项类型是否卡死（"总包中的设计业绩不计"）？面积/合同额门槛是否刚好卡在某梯队？
- **利益交换角度**：定标委员会是否全为业主内部人？评标(外部专家)+定标(全内部)的双层结构是否把主观意志权重做大？规则是否为某家"量身定制"？是否有关系基础才能进场？

每个陷阱标注：**触发条件、后果（废标/扣分/亏损/出局）、我方暴露度、应对动作**。

### 第 5 步：合理/不合理投标响应判定

逐项判断我方（或拟投）响应是否得当：报价策略是否吃透"高罚低奖"公式、业绩是否选对口径、暗标格式是否零失误、合同风险条款是否在签约前争取、联合体是否补对了洼地。指出**合理之处**（继续保持）与**不合理之处**（下次纠正）。

### 第 6 步：输出复盘报告

按 `references/review-report-template.md` 生成 Markdown 报告。如用户要 PDF，用同目录或仓库内的 `md2pdf` 脚本转（中文字体）。报告须包含**保命提醒**章节（废标格式、固定报价填法等"非战之罪出局"高危点）。

## 关键原则

- **区分"舞弊"与"规则性倾斜"**。多数标看不出专家舞弊；真正的偏向写在合法的评分规则里。不要轻易喊"有黑幕"——先做评分反算，用数据说话。
- **陷阱多在界限之外**。硬门槛过了不等于能赢；奖项壁垒、业绩双口径、定标机制才是隐形天花板。
- **诚实标注不可翻盘项**。属于对手护城河、我方结构性洼地的（如他人的装饰协会奖储备），明说"极难翻盘"，不灌鸡汤。
- **废标格式是纯靠格式杀人的陷阱**。有案例仅因暗标"首行未缩进 2 字符"被全票否决——学院派精致排版反而是高危体质。
- **固定总价/创优捆绑/长期驻场是利润绞杀**。把所有材料、政策、工期、获奖风险单方转嫁给设计人的条款要在签约前逐条争取。
- **一切以最终招标文件正式文本及澄清答疑为准**；推算的商务分等须标注为"基于规则的合理估计"。

- `company-profile/` —— **本单位资料参考文件夹（用户填写）**。技能据此对照招标条件给出"对照式"结论。含 `README.md` + `company-profile-template.md`，可另放业绩库/奖项库 CSV/Excel。含敏感资料时建议加入 `.gitignore` 只留模板。
- `references/` —— 方法论与案例：

- `trap-taxonomy.md` —— 九类陷阱全谱 + 排查清单（评奖/业绩/利益交换/废标/报价/合同/创优/驻场/流程）
- `scoring-forensics.md` —— 评分反算取证法（含基准价公式、暗标盲评健康度判据）
- `capability-boundary.md` —— 资质能力边界自评矩阵 + 三情景排位测算法

- `case-library.md` —— 两个实战案例（华电雄安·中标结果复盘；兴业银行·流标重招对比）
