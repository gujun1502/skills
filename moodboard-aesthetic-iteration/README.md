# 意向板审美迭代法 · 使用说明

> Skill 名称：`moodboard-aesthetic-iteration`
> 一句话：用「意向板 + 关键词 + 量化打分 + 跨轮追踪」把效果图的审美质量客观化、可迭代。

---

## 一、这个 skill 是干什么的

设计效果图「好不好看」很主观，导致方案很难有方向地一轮一轮改进。这个 skill 把一套**已验证有效**的方法固化下来：

1. 先设一块**目标意向板（Mood Board）**当标尺；
2. 从意向板提取**关键词三组**当锚点；
3. 用**多套量化框架**给效果图打分，算出「像不像意向 / 质量高不高」；
4. 用**跨轮整改追踪表**客观证明方案一轮比一轮好。

最终每一轮产出一份中文 **markdown 报告 + 内嵌效果图的 PDF**。

核心闭环：

```
意向板(目标) → 关键词(标尺) → 效果图打分(现状) → 差距+整改建议 → 下一轮效果图 → 重新打分(分数应上升)
```

---

## 二、怎么触发

直接用自然语言跟我说，命中以下意图就会自动启用本 skill：

- 「帮我做第 N 轮效果图审美分析」
- 「对照这张意向板给这版方案打分」
- 「这组效果图和意向图差距在哪」
- 「一轮一轮帮我提升效果图质量」
- 「mood board 评分 / 设计迭代评审」

或上传**一组效果图 + 一张/几张意向图**，说明想做客观对照评价即可。

---

## 三、你需要准备什么

| 必备 | 说明 |
|:--|:--|
| 意向板图片 | 概念方案 / 灵感板 / Mood Board，1 张或几张。**这是标尺，没有它无法打分。** |
| 本轮效果图 | 一组待评的效果图（室内/建筑渲染图），按空间归类最好。 |
| 关键词（可选） | 若意向板里已有文字关键词（如「可靠/品质/体验」），告诉我；没有我会从意向板里提取。 |
| 上一轮报告（第2轮起） | 有上一轮的 markdown/PDF 报告，我就能做跨轮整改追踪和分数对比。 |

> 建议把**报告 markdown 和效果图放在同一个文件夹**，渲染 PDF 时图片路径最省事。

---

## 四、完整流程（我会自动按这 8 步走）

```
1. 锁定意向板，提取关键词三组与概念基因
2. 收齐本轮效果图，按空间归类、排重要性
3. 逐图分析：200ms 第一印象 + 6 维打分 + 设计得/失
4. 整体诊断：相似度% / 60-30-10 配色 / 色温 / 三维校准
5.（第2轮起）跨轮整改追踪表 + 分数 delta
6. 综合评分 + 排序提升建议 + 国际对标
7. 写成 markdown 报告
8. 渲染中文 PDF
```

## 五、用到的评分框架（全套固化）

| 框架 | 作用 | 输出 |
|:--|:--|:--|
| 200ms 第一印象 | 抓空间气质 | 一句直觉语义 + 气质吻合度 0–100 |
| Design Critic 6 维 | 单图质量量化 | 色彩和谐/现代性/布局/排版层次/可达性/可用性，各 0–100 |
| 意向相似度 5 维 | 像不像意向板 | 空间气质/灯光/材质/叙事/生物亲和，% |
| Color Harmony | 配色诊断 | 60-30-10 拆色、6 种和声、色温分层偏差 |
| Design Taste 三维 | 疏密/动静/对称 | 对照理想区间打 1–10 |
| 跨轮整改追踪 | 证明在变好 | 上轮问题销账表 + 6 维分数 delta |

详细定义见 `references/scoring-frameworks.md`。

## 六、单独使用 PDF 渲染器

报告写成 markdown 后，一行命令出 PDF（也可单独拿来渲染任何带中文和图片的 markdown）：

```bash
python scripts/md_to_pdf.py "路径/第N轮审美分析报告.md" -o "路径/第N轮审美分析报告.pdf"
```

参数：

| 参数 | 说明 |
|:--|:--|
| 第一个位置参数 | 输入 markdown 文件（必填） |
| `-o / --out` | 输出 PDF 路径（默认同名 `.pdf`） |
| `--base` | 图片基准目录（默认取 markdown 所在目录） |

支持的 markdown 语法：`#/##/###` 标题、`- / *` 列表、`| 表格 |`、`![图注](图片.jpg)` 内嵌图片、`>` 引用块、`---` 分隔线，以及 🟢🟡🔴✅⚠️❌★ 等符号自动转义。

## 八、五条关键原则

1. **标尺先行**：没定意向板和关键词之前不打分——主观评价无法迭代。
2. **口径固定**：同一项目内 6 维评分体系不可中途更换，否则跨轮分数不可比。
3. **每条批评可执行**：「不够高级」是废话；「地面反射度从 85% 降到 30%、改亚光暖灰水磨石」才有用。
4. **概念一以贯之**：核心意象要在每个空间被翻译，而非贴一张图——这是高端方案和「装修」的分水岭。
5. **追踪表是灵魂**：证明方案变好靠上轮问题逐条销账 + 分数上升，不是嘴上说「这版更好」。

## 九、常见问题

**Q：PDF 里某张图显示 `[缺失图片]`？**
A：图片路径没对上。把图片和 md 放同一目录，或用 `--base` 指定图片目录。

**Q：换到 Mac/Linux 能用吗？**
A：渲染器默认用 Windows 自带中文字体（msyh/simhei/simsun）。换系统时在 `scripts/md_to_pdf.py` 顶部 `FONT_CANDIDATES` 里补字体路径即可——脚本已做缺失回退，不会崩。

**Q：依赖什么？**
A：Python 包 `fpdf2`、`Pillow`。`pip install fpdf2 pillow` 即可。

**Q：首轮没有上一轮报告，第五章追踪表怎么办？**
A：首轮自动跳过跨轮追踪，从第二轮起才有。首轮报告就是后续所有轮次的「基准线」。
