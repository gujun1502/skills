# 复盘报告输出模板

> 生成 Markdown；用户要 PDF 时用中文字体脚本转。文末必须有"保命提醒"与免责声明。

```markdown
# 〈项目名〉· 投标复盘与陷阱审计报告

〈业主〉·〈评标办法：综合评估法/评定分离/暗标〉· 编制日期〈YYYY-MM-DD〉

## 摘要
- 最终结果一句话（谁中标、得分、我方/拟投排位）
- 胜负手一句话（差距落在哪个分项——通常是商务标奖项/业绩规则）
- 是否建议投/复盘最大教训一句话

## 一、全生命周期时间轴
公示 → 文件下载窗口 → 答疑/澄清 → 投标截止 → 开标 → 评标 → 定标 → 中标公示 → 结果
（每节点标日期 + 关键参数变化；流标重招的附新旧版 diff 表）

## 二、项目关键参数表
项目名/编号/业主/代理/规模/限价或固定价/保证金/资质门槛/业绩门槛/评标办法/开工节奏/时间节点

## 三、评分结构还原与反算取证（若有公示打分）
- 满分构成（商务/技术暗标/报价）
- 反算总分自洽性 ✓/✗
- 报价分公式复现
- 暗标盲评健康度判断
- 商务分性质（规则化客观分）
- 胜负手定位 + 结论：过程层面有无舞弊 / 规则层面有无倾斜

## 四、投标中存在的问题
文件矛盾、响应不当、被低估的工作量、踩过/差点踩的雷，逐条列。

## 五、我方资质与能力边界（对照 company-profile/）
- 硬卡口打勾表
- 软实力洼地矩阵
- 三情景排位测算 + 诚实结论（极难翻盘项明说）

## 六、陷阱挖掘（九类）
逐类列【触发条件 / 后果 / 我方暴露度 / 应对】。重点突出"界限之外"三角度：
- 评奖角度（协会奖壁垒）
- 业绩角度（双口径/专项卡死/门槛卡梯队）
- 利益交换角度（定标全内部/规则定向/属地关系）

## 七、投标响应合理 / 不合理判定
- 合理之处（继续保持）
- 不合理之处（下次纠正）+ 改进动作

## 八、行动建议（按优先级 P0/P1/P2 + 时限）

## 九、保命提醒（非战之罪高危点）
- 暗标格式零失误（逐页核缩进/字体/脱密/元数据/页数）
- 固定报价填法专人核对
- 澄清期必发问题清单
- 合同签约前必争取条款（预留金释放/违约金/驻场/创优）

---
*本报告基于公开公示与招标文件测算；商务分等推算为基于规则的合理估计，实际以投标人提交材料与评标委员会终判为准。*
```
