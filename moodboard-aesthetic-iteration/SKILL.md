# 意向板审美迭代法（Mood Board Aesthetic Iteration）

把「设定意向板 → 关键词锚定 → 逐轮量化打分 → 客观提升效果图」这套已验证有效的工作流，做成可复用的标准方法。每一轮都产出一份带客观评分的中文审美分析报告（markdown + 内嵌效果图的 PDF）。

## 这套方法解决什么

设计效果图的「好看」很主观，难以迭代。本方法用一块**目标意向板**当标尺、用**关键词三组**当锚点、用**多套量化框架**把「像不像意向」「质量高不高」打成可比的分数，再用**跨轮整改追踪表**客观证明方案一轮一轮在变好。核心闭环：

> 意向板（目标）→ 关键词（标尺）→ 效果图打分（现状）→ 差距与整改建议 → 下一轮效果图 → 重新打分（分数应上升）

## 何时触发

- 用户上传一组效果图 + 一张/几张意向图，要做客观审美对照
- 用户说「做第 N 轮审美分析报告」「给这版方案打分」「对照意向板看差距」
- 用户想把多轮方案迭代的提升过程量化、可视化

## 工作流（复制这份清单逐步推进）

```
审美迭代进度：
- [ ] 1. 锁定意向板，提取关键词三组与概念基因
- [ ] 2. 收齐本轮效果图，按空间归类、排重要性
- [ ] 3. 逐图分析：200ms 第一印象 + 6 维打分 + 设计得/失
- [ ] 4. 整体诊断：相似度 / 60-30-10 / 色温 / 三维校准
- [ ] 5.（第2轮起）跨轮整改追踪表 + 分数 delta
- [ ] 6. 综合评分 + 排序提升建议 + 国际对标
- [ ] 7. 写成 markdown 报告
- [ ] 8. 渲染中文 PDF
```

### 步骤 1 — 锁定意向板，提取关键词

- 找到本项目的**意向板/概念方案/灵感板**图片（Mood Board）。这是整轮的标尺。
- 逐张意向图提取**关键词三组**（每组 3 个左右形容词，如「可靠/品质/体验」这类）和**设计定位**。
- 若有贯穿全案的**概念基因**（单一核心意象），单独拆出它的多层语义——这是后面判断「概念是否一以贯之」的依据。
- 关键词一旦定下，本轮（乃至后续轮次）就不再变，保证标尺稳定。

### 步骤 2 — 收齐效果图并归类

- 列出本轮全部效果图，按空间归类（大堂/中庭/电梯间/办公室……）。
- 按**空间重要性**排序：核心公共空间（大堂、中庭）权重最高，要展开细评；次要空间合并简写。

### 步骤 3 — 逐图分析

对每张图，依 `references/scoring-frameworks.md` 执行：
1. **200ms 第一印象**：一句直觉语义，不分析。
2. **气质吻合度**（0–100）：第一印象 vs 意向关键词。
3. **Design Critic 6 维打分**（0–100，同一项目固定一套口径）。
4. **设计得 / 设计失**：每条都回扣某个意向关键词，整改建议要可执行（给材料/色号/尺寸/参考案例）。

### 步骤 4 — 整体诊断

- **意向相似度评估**（5 维 %，整轮打一次）
- **Color Harmony**：60-30-10 拆色表、色彩和声类型、色温分层偏差表
- **Design Taste 三维校准**：Variance / Motion / Density 对照理想区间

### 步骤 5 — 跨轮整改追踪（第 2 轮起，方法的核心引擎）

- 把上一轮列出的**每条问题**登记进追踪表，标状态（显著改善/部分/微改/未改/无法确认）+ 评价。
- 算**整改率**，列 **6 维分数 delta 表**（上轮→本轮→±）。
- 写**思维变迁**：这一轮相比上一轮在「现代性 / 从图案到肌理 / 概念下沉」上有何跃迁。
- 跨轮分数上升 = 方法奏效的客观证据。

### 步骤 6 — 综合评分与建议

- **最终综合评分表**：概念贯彻 / 色彩 / 材质 / 现代性 / 艺术性 / 可用性 / 施工可行性 → 加权综合分 + 判级。
- **提升建议**：按「影响力 × 可施工性」排序，每条可执行。
- **国际对标**：列 3–5 个标杆案例可借鉴之处。

### 步骤 7 — 写 markdown 报告

严格按 `references/report-structure.md` 的骨架写。图片用 `![图注](文件名.jpg)` 内嵌，路径相对 md 文件所在目录。报告与效果图放同一目录最省事。

### 步骤 8 — 渲染中文 PDF

```bash
python scripts/md_to_pdf.py "路径/第N轮审美分析报告.md" -o "路径/第N轮审美分析报告.pdf"
```

- `--base` 默认取 md 所在目录；图片不在同目录时用 `--base 图片目录` 指定。
- 渲染器自带 Windows 中文字体（msyh/simhei/simsun）、表格、内嵌图片、引用块、emoji 转义。
- 缺图会在 PDF 里标 `[缺失图片]` 而非崩溃——据此回查路径。

## 关键原则

1. **标尺先行**：没定意向板和关键词之前，不要开始打分。主观评价无法迭代。
2. **口径固定**：同一项目内 6 维评分体系不可中途更换，否则跨轮分数不可比。
3. **每条批评可执行**：「不够高级」是废话；「地面反射度从 85% 降到 30%、改亚光暖灰水磨石」才有用。
4. **概念一以贯之**：核心意象要在每个空间被翻译，而非贴一张图。这是高端方案和「装修」的分水岭。
5. **追踪表是灵魂**：客观证明方案变好，靠的是上轮问题逐条销账 + 分数上升，不是嘴上说「这版更好」。

## 文件

- `references/scoring-frameworks.md` — 全套量化评分框架与判级标准（每轮取用）
- `references/report-structure.md` — 报告 markdown 骨架模板
- `scripts/md_to_pdf.py` — 通用 markdown → 中文 PDF 渲染器
