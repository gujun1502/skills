# 方法论 · 解析几何剖面法的哲学与原型

## 一、为什么这套图"有数学哲学的魅力"

普通的示意图是**相似**的：它画一个"看起来像"原理的东西。
这套图是**同一**的：图上每一条线，都是原理的内在方程在某个切面上的**真值**。

> 相似 (resemblance) 是手的工作；同一 (identity) 是理的工作。
> 当观者意识到"这三种截然不同的纹样，竟来自同一组年轮、只是切法不同"，
> 他看到的不再是图画，而是**结构如何生成现象**——这正是数学最深的快感。

三条哲学母题贯穿其中，可在与用户沟通时援引：

1. **庖丁解牛（庄子）**：依乎天理，批大郤，导大窾。剖，要顺着事物**固有的纹理**，
   而不是把网格强加于它。先找到本体的"理"，刀（切面）才知道往哪走。
2. **柏拉图洞穴 / 投影**：我们看到的"相"只是本体在某个面上的影。
   换一个面，影就变；本体始终未变。可视化的任务是把"本体"和"影"同时摆出来。
3. **不变量与表象（康德/群论精神）**：现象千变万化，背后是一个不动的不变量。
   把不变量找出来，让一切表象都由它经一个"操作（切/投/扫）"生成。

## 二、核心断言（搭任何新图都要先满足）

> **同一个参数化本体 + 一族解析操作 = 一族可推导的显象曲线。**

- **本体唯一**：所有视图共享同一组本体参数。改本体 → 所有视图同步变。
- **操作解析**：每个"切法/投影/参数值"都能写成方程或矩阵。
- **显象可导**：每条曲线都有闭式（或可数值精确求解）来源，绝不手绘"形似"。

满足这三条，图就有"理"；缺一条，就退回普通示意图。

## 三、四种原型（按用户的原理选一种套用）

### 原型 A · 剖面型（Sectional）—— 年轮三切法属于此
**本体**：一个实体 / 曲面（圆柱+同心环、球+经纬、晶体+晶面…）。
**操作**：一族平面（点+法向）去截它。
**显象**：截交线（conic section 思维）。
**适合**：任何"分层 / 有内部结构"的对象；想展示"同一结构，不同切面不同纹样"。
**关键解析**：见 `math-derivations.md` 的圆∩平面三式。
**视图组**：在体定位 → 单面纹理 → 爆炸对应 → 规整标注 → 多角度。

### 原型 B · 投影型（Projective）
**本体**：一个高维 / 立体对象（一个 3D 体、一条空间曲线、一个高维流形的低维代理）。
**操作**：向不同方向的平面投影（正交投影矩阵 / 透视）。
**显象**：影子轮廓。经典意象：**同一个立体，正面是圆、侧面是方、顶上是三角**——
"瞎子摸象"与"一体多相"的几何版。
**适合**："看起来矛盾的多个描述其实是同一物的不同投影"。
**关键解析**：点集 P 经投影矩阵 `M_θ` 得 `P' = P · M_θ`；轮廓为投影点云的凸包/边界。
**视图组**：本体居中 → 三/四个方向的影 → 投影方向连线 → 影到本体的"反推"标注。

### 原型 C · 参数扫描型（Parametric Family）
**本体**：一条含参方程（一族曲线/曲面由一个参数生成）。
**操作**：扫这个参数。
**显象**：一族连续变形的曲线。经典：**圆锥曲线**——离心率 e 从 0→1→>1，
同一定义（焦点+准线）连续生成 圆→椭圆→抛物线→双曲线。"四条曲线本是一家"。
**适合**：展示"看似不同的几种东西，是同一律在不同参数下的相位"。
**关键解析**：`r(θ) = l / (1 + e·cosθ)`；扫 e。或李萨如、摆线族、调制正弦族。
**视图组**：参数轴 → 若干离散相位定格 → 连续渐变叠合 → 临界值（分界）标注。

### 原型 D · 变换型（Transformational）
**本体**：一个图形 + 一族变换（旋转/剪切/反演/共形映射）。
**操作**：施加变换群中的元素。
**显象**：同一图形的轨道（orbit）。展示"不变性"——什么变了、什么没变。
**适合**：对称性、守恒、等价类的可视化。
**关键解析**：`P' = T(α)·P`，标出不动点 / 不变量。
**视图组**：原型 → 几个变换像 → 不动点/不变量高亮 → 轨道叠合。

> 多数请求落在 **A 或 C**。拿不准时默认 A（最接近用户喜欢的年轮图），
> 因为它天然带"在体—单面—爆炸—规整—多角度"的完整叙事。

## 四、求导套路（把任意原理逼成可解析的）

1. **找不变量**：问"这原理里，什么是始终不变、生成其余一切的？" → 它就是本体。
2. **找操作族**：问"人们是从哪些不同角度/条件/参数去经验它的？" → 它们是切法。
3. **逐一求显象**：对每个操作，写出本体在其下的方程，**解出闭式**。
   - 解不出 → 要么简化本体（换更可解的几何代理），要么数值精确采样（仍是"算"，非"画"）。
4. **配多视图**：用第三节对应原型的视图组，确保覆盖
   *何处（定位）/ 何象（单面）/ 何以对应（爆炸）/ 如何造（规整）/ 三维何形（多角度）*。
5. **审美收口**：套 `aesthetic-system.md`。

## 五、与用户的对话脚本（开工前必问）

1. "您想解析的**原理/关系**是什么？它的**内在不变结构**（本体）你认为是什么？"
2. "我们用哪几种**切法 / 视角 / 参数**去揭示它？（建议 2–4 种，对比才有张力）"
3. "每种切法，您期望显出什么样的**形态**？（让我把它推成方程）"
4. "成图要偏**自然有机**（带韵律噪声）还是**规整可造**（标尺寸的造型板）？两者都给也行。"

确认这四点，就能照 `example_tree_rings.py` 的骨架换血生成新原理的解析图。
